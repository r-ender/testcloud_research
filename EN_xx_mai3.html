May 2023: Overview IaaS, PaaS and SaaS

This contribution describes the different service-models of cloud computing more extensively. I start with ...

Infrastructure as a Service (IaaS)

The capability provided to the consumer is to provision servers, processing, storage, networks, and other fundamental computing resources where the consumer is able to deploy and run arbitrary software, which can include operating systems and applications. The consumer does not have to manage or control the underlying cloud infrastructure but has control over operating systems, storage, and deployed applications and limited control of selected networking components like host firewalls. Examples for IaaS are Amazon WS (AWS) and Eucalyptus.

The utilized technologies are:
  • Commodity hardware
  • dynamic provisioning of physical resources
  • Virtualization of servers, storage and networks and management of virtual resources
  • Server template technology binding hardware and OS
  • Automated management of the infrastructure enabling self-service of costumers

Platform as a Service (PaaS)

The capability provided to the consumer is to install onto the cloud infrastructure consumer-created or acquired applications created using programming languages, libraries, services, databases, development environments (IDE) and tools supported by the user respectively provider. The consumer does not manage or control the underlying cloud infrastructure like network, servers, operating systems, or storage, but has control over the deployed applications and configuration settings for the application-hosting environment. This principle is called serverless computing, the user can write software code without the need to take care of underlying infrastructure. Examples for PaaS are Microsoft Azure's Microservice Fabric, Amazon's FaaS and Google's Application Engine.

The utilized technologies are:
  • Service Oriented Architecture (SOA)
  • Services that can be used in your application
  • Scalable application middleware, e.g. databases, file systems
  • Special programming models and interfaces Automation

Software as a Service (SaaS)

The provided capability are consumer or industrial applications to individual or enterprise users, running on a cloud infrastructure. The applications are accessible from various client devices through either a web browser, called thin client, or a program interface, called API. The consumer does neither manage or control the underlying cloud infrastructure including network, servers, operating systems, storage, nor even individual application capabilities, maybe excepted limited user-specific application settings. Examples for SaaS are Salesforce, Google, Dropbox.

The utilized technologies are:
  • Standard and cloud native applications
  • Autoscaling
  • Multitenancy
  • Variety of user interfaces

As you can see the level abstraction increases from Iaas to SaaS.
